## Clean-Room Protocol

For permissive repos, we can read source, compare CLI ergonomics, and adapt
ideas when the resulting implementation is independently written in this codebase.
Keep attribution in docs when a feature is materially inspired by another tool.

The goal is not license timidity. If another project has technically excellent
code or behavior, record that clearly. Then choose the right incorporation path:
direct Rust port with attribution for permissive material, independent
reimplementation from specs/fixtures for restricted material, an explicitly
licensed optional adapter when we accept that license, or behavior-test-only
coverage when direct reuse would make the public core harder to ship.

For quarantined repos:

1. Do not copy code, schemas, examples, fixture JSON, prompts, docs prose, or
   command implementations into `powerbi-cli`.
2. Use them only to identify external behavior: command categories, user
   workflows, validation expectations, and gaps users care about.
3. Implement from primary sources where possible: Microsoft schemas, Desktop
   round-trips generated by us, and our own sample reports.
4. When a quarantined project suggests a behavior, recreate the behavior from
   first principles using our own Desktop oracle fixture before adding code.
5. Keep quarantined repositories out of commits, tests, release artifacts, and
   generated examples.

If we need a strict clean-room pass, split the work:

- Researcher: reads quarantined material and writes a high-level requirements
  note with no code, no JSON fragments, and no low-level structural recipes.
- Implementer: uses only the requirements note, Microsoft schemas/docs, and
  newly generated Power BI Desktop fixtures.

## Competitive Implications

`MinaSaad1/pbi-cli` is the closest permissive overlap. It already targets agents,
semantic models, PBIR report files, visual operations, filters, formatting,
bookmarks, themes, and custom visuals. We should not pretend `powerbi-cli` is
alone in the space.

The strongest niche for this project is still distinct:

- cross-platform single-binary Rust CLI;
- deterministic schema-to-PBIP/PBIR/TMDL generation;
- offline-safe dummy-data and schema-first handoff for locked-down corporate
  environments;
- oracle-driven golden fixtures generated by our own Power BI Desktop round
  trips;
- agent-friendly JSON output, validation, and reproducible report builds.

That means we should prioritize a compiler-like workflow over a broad interactive
Desktop automation layer.
